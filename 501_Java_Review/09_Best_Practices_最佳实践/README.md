# Java最佳实践

## 1. 编码规范

### 1.1 命名规范
- **包名**：全小写，使用域名反转（com.example.project）
- **类名**：大驼峰式（HelloWorld）
- **方法名**：小驼峰式（getUserInfo）
- **变量名**：小驼峰式（userName）
- **常量名**：全大写，下划线分隔（MAX_VALUE）
- **接口名**：大驼峰式，通常以able或ible结尾（Runnable）

### 1.2 代码格式
- 使用4个空格缩进，不使用制表符
- 每行代码不超过120个字符
- 运算符前后加空格
- 逗号后加空格
- 大括号与关键字在同一行

### 1.3 注释规范
- **类注释**：说明类的作用、作者、创建时间
- **方法注释**：说明方法的作用、参数、返回值、异常
- **行注释**：说明代码的逻辑和意图
- **避免不必要的注释**：代码本身应该是自解释的

### 1.4 示例代码
```java
/**
 * 用户服务类
 * @author 张三
 * @date 2023-01-01
 */
public class UserService {
    
    /**
     * 获取用户信息
     * @param userId 用户ID
     * @return 用户信息
     * @throws UserNotFoundException 用户不存在异常
     */
```

## 2. 性能优化

### 2.1 内存优化
- 减少对象的创建
- 避免内存泄漏
- 合理设置堆内存大小
- 使用对象池

### 2.2 垃圾回收优化
- 选择合适的垃圾回收器
- 合理设置新生代和老年代的比例
- 减少Full GC的次数
- 避免显式调用System.gc()

### 2.3 代码优化
- 避免使用finalize()方法
- 避免使用显式的锁
- 合理使用缓存
- 避免在循环中创建对象

### 2.4 并发优化
- 合理使用线程池
- 避免死锁
- 合理使用volatile关键字
- 合理使用原子类

## 11. 常见问题

### 11.1 空指针异常
- 原因：调用了null对象的方法
- 解决：检查对象是否为null

### 11.2 内存泄漏
- 原因：对象不再使用但未被回收
- 解决：避免静态集合持有对象引用

### 11.3 死锁
- 原因：多个线程相互等待对方释放锁
- 解决：按顺序获取锁

### 11.4 性能问题
- 原因：代码效率低、资源不足
- 解决：优化代码、增加资源
