## 2天落地执行计划

### Day 1：核心原理学习（浅尝概念，30分钟背完）

#### 一、先搞懂：向量库是什么？
- 大模型/AI系统的**专用数据库**，只存**文本向量(Embedding)**，专门做**语义相似检索**
- 比如RAG中：用户提问 → 搜知识库中语义最接近的内容

#### 二、Qdrant / Milvus 核心概念（必记）
1. **向量 (Vector)**：文字转成的数字数组（Embedding结果）
2. **集合 (Collection)**：相当于普通数据库的「表」
3. **入库**：把切片后的文档向量存进库
4. **检索**：输入问题向量，找出**语义最相似**的文本
5. **相似度**：用**余弦相似度**判断文本语义远近

#### 三、二选一学习策略（最省力）
1. **Qdrant（掌握级）**
   - 轻量、速度快、纯Python可运行、**无需复杂部署**
   - 个人/小型项目首选，适配你的Demo
2. **Milvus（了解级）**
   - 大厂/企业级生产常用、功能更强、部署稍重
   - 只需记住：分布式向量库，用于大规模RAG系统

#### 四、一句话口诀
- 向量库 = 存向量 + 搜相似文本 → RAG系统的核心组件

## 向量库的工作原理

### 1. 向量生成
- 将文本、图像、音频等非结构化数据转换为向量表示
- 使用预训练的Embedding模型，如sentence-transformers、BERT等

### 2. 向量存储
- 将生成的向量存储到向量库中
- 支持批量插入和更新

### 3. 向量检索
- 输入查询向量，找出语义最相似的向量
- 使用近似最近邻（ANN）算法，如HNSW、IVF等

### 4. 向量相似度计算
- 使用余弦相似度、欧氏距离等计算向量之间的相似度
- 相似度越高，语义越接近

## 常见的向量库类型

### 1. 轻量级向量库
- Qdrant: 轻量、本地一键运行、CPU兼容
- FAISS: Facebook开源的向量库，支持GPU加速
- Annoy: Spotify开源的向量库，适合大规模数据

### 2. 企业级向量库
- Milvus: 企业级生产常用、功能更强、部署稍重
- Pinecone: 云原生向量库，支持分布式部署
- Weaviate: 开源的向量数据库，支持多模态数据

## 向量库的性能优化

### 1. 索引优化
- 选择合适的索引类型，如HNSW、IVF等
- 调整索引参数，如nlist、efConstruction等

### 2. 硬件优化
- 使用GPU加速向量检索
- 增加内存和CPU资源

### 3. 数据优化
- 减少向量维度，如使用PCA降维
- 批量插入和更新数据
