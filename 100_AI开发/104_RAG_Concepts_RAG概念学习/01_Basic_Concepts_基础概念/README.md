# RAG基础概念

## 什么是RAG？

RAG（Retrieval-Augmented Generation）是一种结合了**检索**和**生成**的AI技术，通过从外部知识库中检索相关信息，然后将这些信息与用户查询一起输入到语言模型中，以生成更准确、更相关的回答。

## RAG的核心价值

1. **解决知识过时问题**：通过检索最新的外部文档，弥补大模型训练数据的时效性问题
2. **减少幻觉**：基于真实文档生成回答，降低大模型编造信息的可能性
3. **提高回答准确性**：参考相关文档内容，生成更符合事实的回答
4. **领域知识增强**：快速融入特定领域的专业知识，无需重新训练大模型

## RAG工作流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  文档加载       │────>│  文档切片       │────>│  向量编码       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                                      │
                                      ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  用户查询       │────>│  语义检索       │<────│  向量存储       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                                      │
                                      ▼
                            ┌─────────────────┐
                            │  增强生成       │
                            └─────────────────┘
                                      │
                                      ▼
                            ┌─────────────────┐
                            │  生成回答       │
                            └─────────────────┘
```

## RAG与传统LLM的区别

| 特性 | 传统LLM | RAG |
|------|---------|-----|
| 知识来源 | 训练数据 | 外部文档 + 训练数据 |
| 知识时效性 | 截止训练时间 | 实时更新的文档 |
| 回答准确性 | 可能产生幻觉 | 基于文档，更准确 |
| 领域适应性 | 需要微调 | 直接使用领域文档 |
| 可解释性 | 低 | 高（可引用来源） |

## RAG的应用场景

1. **知识库问答**：企业知识库、产品文档、学术资料
2. **客户服务**：智能客服、故障排查、FAQ系统
3. **内容创作**：辅助写作、内容摘要、研究助手
4. **教育领域**：智能辅导、作业辅助、个性化学习

## 核心概念

1. **文档加载**：将各种格式的文档加载到系统中
2. **文档切片**：将长文档切分为较小的文本块
3. **向量编码**：将文本转换为数字向量
4. **向量存储**：将向量存储到向量数据库中
5. **语义检索**：根据用户查询检索相关文本块
6. **增强生成**：基于检索结果生成回答

## 技术栈选择

- **文档处理**：pypdf、python-docx、langchain.document_loaders
- **文本切片**：langchain.text_splitter
- **Embedding**：sentence-transformers、OpenAI Embeddings
- **向量库**：Chroma、Qdrant、Milvus、FAISS
- **语言模型**：GPT-3.5/4、Claude、Qwen、Llama
