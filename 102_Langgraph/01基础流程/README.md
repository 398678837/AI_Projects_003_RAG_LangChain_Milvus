# LangGraph基础流程

本示例展示了如何使用LangGraph创建一个基础的流程，包含2个节点+条件边判断问题类型。

## 功能说明

- 使用StateGraph创建状态图
- 包含3个节点：分析问题类型、回答技术问题、回答非技术问题
- 使用条件边根据问题类型选择路径
- 适配Qwen等通用大模型
- 支持批量问答和交互式问答

## 代码结构

```python
# 1. 配置API密钥和大模型
# 2. 定义状态类型
# 3. 定义节点函数
# 4. 定义条件边函数
# 5. 创建状态图
# 6. 测试流程
```

## 配置说明

1. **API密钥配置**：
   - 在环境变量中设置 `API_KEY`
   - 或直接在代码中修改 `api_key` 变量

2. **大模型配置**：
   - 默认使用Qwen模型的API地址
   - 可以根据需要修改 `base_url`

## 运行示例

1. 安装依赖：
   ```bash
   pip install -r ../../requirements.txt
   ```

2. 运行示例：
   ```bash
   python demo.py
   ```

3. 交互模式：
   - 输入问题，按回车获取回答
   - 输入'quit'退出

## 学习要点

- **StateGraph**：用于创建状态图，定义流程
- **节点**：流程中的各个处理步骤
- **条件边**：根据状态选择不同的路径
- **状态管理**：在流程中传递和修改状态

## 扩展建议

- 添加更多节点，构建更复杂的流程
- 实现状态持久化，保存流程状态
- 集成外部工具，如搜索引擎、数据库等
- 优化条件判断逻辑，提高问题分类准确率
