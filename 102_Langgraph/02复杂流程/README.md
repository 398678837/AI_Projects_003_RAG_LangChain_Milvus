# LangGraph复杂流程

本示例展示了如何使用LangGraph创建更复杂的多节点流程，包含5个节点和多个条件边。

## 功能说明

- 使用StateGraph创建复杂的状态图
- 包含5个节点：分析问题类型、分析问题内容、回答技术问题、回答非技术问题、检索相关信息
- 使用多个条件边实现复杂的路径选择
- 适配Qwen等通用大模型
- 支持批量问答和交互式问答

## 代码结构

```python
# 1. 配置API密钥和大模型
# 2. 定义状态类型
# 3. 定义节点函数
# 4. 定义条件边函数
# 5. 创建状态图
# 6. 测试流程
```

## 配置说明

1. **API密钥配置**：
   - 在环境变量中设置 `API_KEY`
   - 或直接在代码中修改 `api_key` 变量

2. **大模型配置**：
   - 默认使用Qwen模型的API地址
   - 可以根据需要修改 `base_url`

## 运行示例

1. 安装依赖：
   ```bash
   pip install -r ../../requirements.txt
   ```

2. 运行示例：
   ```bash
   python demo.py
   ```

3. 交互模式：
   - 输入问题，按回车获取回答
   - 输入'quit'退出

## 学习要点

- **复杂状态图**：如何创建包含多个节点和条件边的复杂流程
- **多路径选择**：如何根据不同条件选择不同的处理路径
- **节点间数据传递**：如何在节点间传递和修改状态
- **模块化设计**：如何将复杂任务分解为多个子任务
