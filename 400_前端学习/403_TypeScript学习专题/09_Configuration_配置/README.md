# 09_Configuration_配置

## 学习目标
- 理解TypeScript配置的基本概念
- 掌握tsconfig.json的配置选项
- 熟悉不同项目类型的配置方式
- 学会使用ESLint和Prettier进行代码风格检查

## 知识点总结
1. **tsconfig.json 基本配置**：基本的TypeScript编译选项
2. **tsconfig.json 完整配置**：包含所有常用的编译选项
3. **tsconfig.node.json 配置**：用于配置文件的TypeScript配置
4. **tsconfig.json 用于 React 项目**：React项目的TypeScript配置
5. **tsconfig.json 用于 Next.js 项目**：Next.js项目的TypeScript配置
6. **tsconfig.json 用于 Node.js 项目**：Node.js项目的TypeScript配置
7. **tslint.json 配置**：TypeScript代码风格检查配置
8. **eslintrc.json 配置**：ESLint代码风格检查配置
9. **.prettierrc 配置**：Prettier代码格式化配置
10. **package.json 中的脚本**：配置项目脚本
11. **ts-node 配置**：ts-node的配置选项
12. **环境变量配置**：使用.env文件配置环境变量
13. **类型声明文件配置**：配置类型声明文件的路径
14. **路径别名配置**：配置模块导入的路径别名
15. **增量编译配置**：启用增量编译提高构建速度
16. **复合项目配置**：配置多项目的依赖关系
17. **模块解析配置**：配置模块解析策略
18. **严格模式配置**：启用TypeScript的严格模式
19. **输出配置**：配置编译输出的目录和选项
20. **包含和排除配置**：配置需要编译的文件和排除的文件
