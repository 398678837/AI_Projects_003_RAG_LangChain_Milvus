# 持久化 - 木马持久化原理

1. **理解持久化的基本原理**：掌握木马如何在系统中持久存在

3. **掌握持久化的实现方法**：学习如何使用Windows API和命令行实现持久化
4. **了解持久化的安全风险**：认识持久化技术的潜在危害
5. **掌握持久化的检测和防御方法**：学习如何检测和防御持久化木马

## 核心概念

### 1. 持久化的定义

持久化是指木马在系统重启后仍然能够自动运行的技术。在木马技术中，持久化是确保木马长期存在于目标系统中的关键技术，它可以使木马在系统重启后继续执行恶意活动。

### 2. 常见的持久化技术

常见的持久化技术包括：

- **启动项**：将木马添加到系统启动项中，使系统启动时自动运行
- **服务**：将木马注册为系统服务，使系统启动时自动运行
- **计划任务**：创建计划任务，在特定时间或事件触发时运行木马
- **WMI事件订阅**：使用WMI事件订阅机制，在特定事件发生时运行木马
- **DLL劫持**：利用DLL加载顺序，劫持合法DLL的加载
- **启动文件夹**：将木马快捷方式添加到启动文件夹中，使系统启动时自动运行
- **引导过程**：修改系统引导过程，在系统启动早期运行木马
- **固件感染**：感染系统固件，使木马在系统重装后仍然存在

### 3. 持久化的基本原理

持久化的基本原理是：

1. **选择合适的持久化位置**：根据系统环境和安全策略选择合适的持久化位置
2. **添加持久化项**：在选定的位置添加木马的启动信息
3. **确保权限**：确保有足够的权限添加持久化项
4. **隐藏持久化项**：隐藏持久化项，避免被检测

### 4. 持久化的安全风险

持久化技术的安全风险主要包括：

- **长期存在**：木马可以在系统中长期存在，持续执行恶意活动
- **难以清除**：持久化项可能隐藏得很深，难以被发现和清除
- **系统重启后仍然运行**：即使系统重启，木马仍然能够自动运行
- **权限提升**：通过持久化，木马可以在系统启动时以较高权限运行

### 5. 持久化的检测和防御方法

检测和防御持久化的方法主要包括：

- **启动项检查**：检查系统启动项，查找可疑的启动项
- **服务检查**：检查系统服务，查找可疑的服务
- **计划任务检查**：检查系统计划任务，查找可疑的计划任务
- **启动文件夹检查**：检查启动文件夹，查找可疑的快捷方式
- **WMI事件订阅检查**：检查WMI事件订阅，查找可疑的订阅
- **系统文件检查**：检查系统文件，查找被修改的文件
- **行为分析**：分析程序的行为，识别异常

## 技术详解

### 1. 启动项持久化

启动项持久化是将木马添加到系统启动项中，使系统启动时自动运行。启动项主要包括：

- **注册表启动项**：
  - `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`：当前用户的启动项
  - `HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run`：所有用户的启动项
  - `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\RunOnce`：当前用户的一次性启动项
  - `HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\RunOnce`：所有用户的一次性启动项

- **启动文件夹**：
  - 当前用户启动文件夹：`%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`
  - 所有用户启动文件夹：`%ProgramData%\Microsoft\Windows\Start Menu\Programs\StartUp`

**实现方法**：
- 使用注册表API添加启动项
- 在启动文件夹中创建快捷方式

### 2. 服务持久化

服务持久化是将木马注册为系统服务，使系统启动时自动运行。服务具有以下特点：

- **自动启动**：可以设置为系统启动时自动启动
- **后台运行**：在后台运行，不需要用户登录
- **高权限**：可以以系统权限运行

**实现方法**：
- 使用`sc`命令创建服务
- 使用Windows API创建服务

### 3. 计划任务持久化

计划任务持久化是创建计划任务，在特定时间或事件触发时运行木马。计划任务具有以下特点：

- **灵活的触发条件**：可以设置为在特定时间、系统启动、用户登录等事件触发
- **后台运行**：在后台运行，不需要用户交互
- **较高权限**：可以以较高权限运行

**实现方法**：
- 使用`schtasks`命令创建计划任务
- 使用任务计划程序GUI创建计划任务
- 使用Windows API创建计划任务

### 4. WMI事件订阅持久化

WMI事件订阅持久化是使用WMI事件订阅机制，在特定事件发生时运行木马。WMI事件订阅具有以下特点：

- **隐蔽性高**：不容易被发现
- **灵活的触发条件**：可以设置为在各种系统事件发生时触发
- **后台运行**：在后台运行，不需要用户交互

**实现方法**：
- 使用WMI命令行工具创建事件订阅
- 使用PowerShell创建事件订阅
- 使用Windows API创建事件订阅

### 5. DLL劫持持久化

DLL劫持持久化是利用DLL加载顺序，劫持合法DLL的加载。DLL劫持具有以下特点：

- **隐蔽性高**：利用合法程序的DLL加载机制
- **不需要修改系统设置**：只需要将恶意DLL放置在合适的位置
- **与合法程序一起运行**：在合法程序运行时被加载

**实现方法**：
- 识别目标程序依赖的DLL
- 创建同名的恶意DLL
- 将恶意DLL放置在程序的DLL搜索路径中

### 6. 引导过程持久化

引导过程持久化是修改系统引导过程，在系统启动早期运行木马。引导过程持久化具有以下特点：

- **启动早**：在系统启动早期运行，甚至在操作系统加载之前
- **权限高**：可以以最高权限运行
- **难以检测**：传统的安全工具可能无法检测

**实现方法**：
- 修改MBR（主引导记录）
- 修改VBR（卷引导记录）
- 修改引导配置数据（BCD）

### 7. 固件感染持久化

固件感染持久化是感染系统固件，使木马在系统重装后仍然存在。固件感染具有以下特点：

- **持久性极强**：即使重装操作系统，木马仍然存在
- **难以检测**：传统的安全工具无法检测
- **难以清除**：需要特殊的工具和方法才能清除

**实现方法**：
- 感染BIOS/UEFI固件
- 感染硬盘固件
- 感染其他硬件固件

`Persistence` 类实现了持久化技术的演示，包含以下核心功能：

## 常见问题与解决方案

### 问题一：权限不足

**解决方案**：
- 以管理员权限运行程序
- 使用UAC提权
- 利用系统漏洞获取权限

### 问题二：持久化项被检测

**解决方案**：
- 使用更隐蔽的持久化方法，如WMI事件订阅
- 隐藏持久化项，如使用随机名称
- 伪装持久化项，使其看起来像合法程序
- 多重持久化，使用多种持久化方法

### 问题三：持久化项被清除

**解决方案**：
- 定期检查持久化项，确保其存在
- 实现自我修复机制，在持久化项被清除后重新添加
- 使用难以清除的持久化方法，如固件感染

### 问题四：系统升级后持久化失效

**解决方案**：
- 选择受系统升级影响较小的持久化方法
- 监控系统升级，在升级后重新添加持久化项
- 使用多个持久化位置，确保至少有一个能够生效

## 扩展与进阶

### 1. 高级持久化技术

- **无文件持久化**：不使用文件系统，直接在内存或注册表中实现持久化
- **内存持久化**：将代码存储在内存中，系统重启后通过其他方式重新加载
- **网络持久化**：从网络获取代码，不依赖本地存储
- **多阶段持久化**：使用多个阶段的持久化方法，确保长期存在

- **定期检查**：定期检查系统中的持久化项
- **权限管理**：限制程序的权限，防止未授权的持久化
- **行为分析**：分析程序的行为，识别异常的持久化行为
- **系统完整性监控**：监控系统文件和设置的变化
- **安全启动**：启用安全启动，防止未签名的代码在系统启动时运行
